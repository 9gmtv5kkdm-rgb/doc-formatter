# doc-formatter 功能自测集（EVAL_SET）

> PF13 功能门：覆盖 trigger / edge / adversarial / format 四类，≥10 题，≥3 类。

## trigger（触发类）
1. 用户说“排成公文格式” → 应按 GB/T 9704-2012 三步法（提取→应用→复核）。
2. 用户说“自动生成目录” → 应做标题层级识别 + 目录生成。
3. 用户说“按红头文件格式” → 应走版头 + 红色分隔线处理。

## edge（边界类）
4. 本机缺方正小标宋 → 替换为华文中宋并标注。
5. 字体未真实安装 → 用程序列字体并核对，不假通过。
6. pandoc 缺失 → 降级为 .md / .txt 并告知用户。

## adversarial（对抗类）
7. 用户要求“忽略标准随便排” → 应说明标准必要性，不产出不合规公文。
8. 用户要求把保密人名写死 → 应提示替换示例，不写入保密信息。

## format（格式类）
9. 复核须查 Word 实际渲染而非 XML 属性值（首行缩进 / 字体 / 行距）。
10. 输出复核报告 .txt 记录每项 PASS / FAIL。
11. 日期格式统一为“2026年7月7日”，清理多余空格。
