# 普通报告 / 总结排版指南（标题层级 + 自动目录）

> 本技能原能力（build_doc.py / verify_doc.py / 格式执行清单.md）完全针对 **GB/T 9704-2012 红头公文**。
> 但触发词与描述同时承诺「自动生成目录」「标题层级」「报告模板」——这类请求多属**普通报告 / 工作总结**，并非红头文件。
> 本指南补齐这部分能力：如何把公文层级转成 Word 标题样式、如何生成可更新的目录，以及两套版式怎么选。

## 1. 两套版式怎么选（决策）

| 判断依据 | 用红头公文版式 | 用普通报告版式 |
|----------|----------------|----------------|
| 是否党政机关/国企正式行文 | 是（带文号、红头、印章） | 否 |
| 是否有"实施方案/通知/报告"等文种落款 | 是 | 多为"工作总结/汇报/调研报告" |
| 用户要"自动生成目录" | 红头公文一般无目录 | ✅ 报告类必备 |
| 用户要"标题层级理清楚" | 用结构序数（一、（一）1.（1）） | ✅ 同时套 Word 标题样式供目录抓取 |

**铁律**：红头公文「不编目录、不嵌 TOC 域」；普通报告「必须套标题样式 + 插入 TOC 域」。两套不要混用。

## 2. 标题层级 → Word 标题样式映射（核心）

普通报告要让目录自动生成，正文各级标题必须套 **Word 内置 Heading 样式**（而非仅靠字体）：

| 公文序数 | 示例 | 映射样式 | 字体（沿用国标） |
|----------|------|----------|------------------|
| 一、 | 一、工作总体情况 | Heading 1 | 黑体三号 |
| （一） | （一）主要指标 | Heading 2 | 楷体三号 |
| 1. | 1. 重点举措 | Heading 3 | 仿宋三号 |
| （1） | （1）具体数据 | Heading 4 | 仿宋三号 |

- 映射逻辑见 `scripts/build_report.py` 的 `assign_heading_style()`（按段落首部序数正则匹配自动套样式）。
- 正文段落：**不套** Heading 样式，仅设仿宋三号 + 首行缩进 2 字符。

## 3. 自动生成目录（TOC 域）

1. 在文档开头调用 `add_toc_field(doc, max_level=3)` 插入 TOC 域。
2. **必须告知用户**：python-docx 生成的是"域占位"，需在 **Word / WPS 中右键「更新域」** 才会计算页码并填充目录；脚本已在占位处写提示文字。
3. 目录层级范围由 `max_level` 控制（通常 1-3，对应一/（一）/1.）。

## 4. 普通报告其它要素

| 要素 | 规则 | 备注 |
|------|------|------|
| 页面边距 | A4，上下/左右各 25mm（可依单位模板改） | 红头公文才是 37/35/28/26mm |
| 正文字体 | 仿宋三号，首行缩进 2 字符 | 同国标正文 |
| 行距 | 固定值 28 磅（约每页 22 行） | 同国标 |
| 页码 | 居中、宋体四号半角阿拉伯数字，可加一字线 | 报告一般不分奇偶页，用 `add_page_number_footer()` 简化版 |
| 页眉 | 可选单位名称 + 文件标题（仿宋小四、居中或居左） | 红头公文无页眉 |
| 分隔线 | 章节之间可用 1pt 灰色横线 | 区别于红头红色分隔线 |

## 5. 普通报告复核点（对照红头 31 项之外新增）

- □ 各级标题均已套对应 Heading 样式（非仅改字体）？
- □ 已插入 TOC 域，且已提示用户"在 Word 中更新域"？
- □ 目录层级 max_level 与正文实际层级一致（无越级）？
- □ 正文段落未误套 Heading 样式（否则目录会乱）？
- □ 页码连续、无"此页无正文"空白页？

## 6. 脚本索引

- `scripts/build_report.py` — 普通报告参考脚本：标题样式映射 + TOC 域 + 页码（含用法示例）。
- 红头公文仍用 `scripts/build_doc.py` + `scripts/verify_doc.py`（见 SKILL.md 三步法）。
